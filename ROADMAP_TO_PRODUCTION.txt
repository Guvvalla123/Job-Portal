================================================================================
              JOB PORTAL - ROADMAP TO PRODUCTION
                    (Real Portal Upgrade)
================================================================================

This document outlines the work needed to evolve the current MVP into a
production-ready job portal. Work in phases; each phase is shippable.

================================================================================
PHASE 1: FOUNDATION & CRITICAL GAPS (Do First)
================================================================================

1.1 SMTP & EMAIL
    - [ ] Configure SMTP in .env
    - [ ] Forgot password emails (backend ready)
    - [ ] Job alert emails (backend ready)
    - [ ] Email verification on signup (optional but recommended)

1.2 RESUME/CV UPLOAD
    - [ ] Backend: Resume upload endpoint (Cloudinary) - User model has resumeUrl
    - [ ] Frontend: Resume upload in candidate profile + apply flow
    - [ ] Recruiter: View/download resume from applicant card

1.3 RECRUITER IMPROVEMENTS (parity with candidate)
    - [ ] Edit company (name, website, location, description)
    - [ ] Company logo upload
    - [ ] Recruiter dashboard tabs (Overview, Companies, Jobs, Applications)
    - [ ] Reactivate deactivated jobs
    - [ ] Applicant filters (by status)
    - [ ] View full candidate profile from applicant card

1.4 PAGINATION & LOADING
    - [ ] Pagination on Jobs page (backend has it, frontend may need UI)
    - [ ] Pagination on Companies page
    - [ ] Pagination on Admin lists (users, jobs, applications)
    - [ ] Loading skeletons where missing

================================================================================
PHASE 2: SEARCH & DISCOVERY
================================================================================

2.1 JOB SEARCH
    - [ ] Salary range filter (min/max)
    - [ ] Remote/hybrid/onsite filter
    - [ ] Skills filter (multi-select)
    - [ ] Sort by: date, salary, relevance
    - [ ] Better text search (MongoDB $text or consider Elasticsearch later)

2.2 COMPANY SEARCH
    - [ ] Filter by location, industry (if we add industry)
    - [ ] Sort options

2.3 CANDIDATE DISCOVERY (for recruiters)
    - [ ] Search candidates by skills, location (public profile search)
    - [ ] Recruiter can browse/search candidate profiles (with privacy controls)

================================================================================
PHASE 3: ATS-LITE (Applicant Tracking)
================================================================================

3.1 APPLICATION PIPELINE
    - [ ] Custom pipeline stages (e.g. Screening → Phone → Onsite → Offer)
    - [ ] Drag-and-drop to move applicants between stages
    - [ ] Bulk actions (shortlist multiple, reject multiple)

3.2 NOTES & COLLABORATION
    - [ ] Recruiter notes on applications (private, per application)
    - [ ] Internal rating/score on applicants
    - [ ] Activity log per application

3.3 INTERVIEW SCHEDULING (basic)
    - [ ] Add "Interview date" field to application
    - [ ] Calendar view of scheduled interviews (optional)
    - [ ] Email candidate when interview is scheduled

================================================================================
PHASE 4: AUTHENTICATION & SECURITY
================================================================================

4.1 OAUTH
    - [ ] Google Sign-In
    - [ ] LinkedIn Sign-In (valuable for job portals)

4.2 SECURITY HARDENING
    - [ ] Rate limiting per endpoint (tune existing)
    - [ ] CSRF protection
    - [ ] Audit log for sensitive actions (admin)
    - [ ] Session management (revoke tokens, logout all devices)

4.3 EMAIL VERIFICATION
    - [ ] Verify email on signup
    - [ ] Resend verification email

================================================================================
PHASE 5: COMPANY BRANDING & UX
================================================================================

5.1 COMPANY PAGES
    - [ ] Company logo on public page
    - [ ] Company banner/cover image
    - [ ] "Why join us" / culture section
    - [ ] Benefits & perks
    - [ ] Team size, founded year (optional)

5.2 JOB PAGES
    - [ ] Apply with one click (prefill from profile)
    - [ ] "Similar jobs" section on job detail
    - [ ] Share job (social links)
    - [ ] Report job (flag inappropriate)

5.3 PROFILE & SETTINGS
    - [ ] Theme toggle in settings/profile dropdown
    - [ ] Notification preferences (email on/off per type)
    - [ ] Privacy settings (profile visibility)
    - [ ] Export my data (GDPR-style)

================================================================================
PHASE 6: MONETIZATION (Optional - for business)
================================================================================

6.1 JOB POSTING
    - [ ] Free tier: X jobs per month
    - [ ] Paid: Featured jobs, boost visibility
    - [ ] Stripe/payment integration

6.2 RECRUITER SUBSCRIPTION
    - [ ] Plans: Basic, Pro, Enterprise
    - [ ] Limits: jobs, applications, team members

6.3 CANDIDATE PREMIUM (optional)
    - [ ] See who viewed profile
    - [ ] Apply insights
    - [ ] Priority support

================================================================================
PHASE 7: SCALE & RELIABILITY
================================================================================

7.1 INFRASTRUCTURE
    - [ ] Environment: staging vs production
    - [ ] Database indexes for performance
    - [ ] Redis for session/cache (optional)
    - [ ] CDN for static assets

7.2 MONITORING
    - [ ] Error tracking (Sentry)
    - [ ] Logging (structured logs)
    - [ ] Uptime monitoring

7.3 TESTING
    - [ ] Unit tests (critical paths)
    - [ ] API integration tests
    - [ ] E2E tests (Playwright/Cypress)

================================================================================
RECOMMENDED ORDER TO START
================================================================================

Week 1-2:  Phase 1.1 (SMTP) + 1.2 (Resume upload) + 1.3 (Recruiter improvements)
Week 3:    Phase 1.4 (Pagination) + Phase 2.1 (Job search filters)
Week 4:    Phase 3.1 (Pipeline) + 3.2 (Notes)
Week 5+:   Phase 4 (OAuth, security), Phase 5 (Branding), etc.

================================================================================
