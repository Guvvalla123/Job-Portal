JOB PORTAL MERN — PROJECT OVERVIEW
Last Updated: March 27, 2026
════════════════════════════════════════

WHAT IT IS
──────────
Job Portal MERN is a full-stack job board similar in spirit to Naukri: candidates
search and apply to jobs, manage profiles and PDF resumes, and configure email
alerts; recruiters and admins manage companies, job postings, and application
pipelines with analytics; admins moderate the platform and review audit logs.

──────────

          BullMQ ^5.71.0 + ioredis ^5.4.1 + Winston ^3.17.0 + Zod ^4.3.6 +
          Nodemailer ^8.0.2 + bcryptjs ^3.0.3 + Helmet ^8.1.0 + express-rate-limit ^

          backend/package.json).
Frontend: React ^19.2.0 + Vite ^7.3.1 + Tailwind CSS ^4.2.1 (@tailwindcss/vite) +
          TanStack React Query ^5.90.21 + React Router ^7.13.1 +

          Recharts ^3.8.1 + axios ^1.13.6 + sonner + react-helmet-async +
          react-error-boundary + prop-types (see frontend/package.json).

USER ROLES
──────────
Candidate: Browse jobs and companies; apply (once per job); dashboard; profile
           with skills/experience/projects/education; profile image; PDF resume;
           saved jobs; notifications; job alerts (IMMEDIATE/DAILY/WEEKLY).
Recruiter: Companies (CRUD, logo); jobs (CRUD, drafts, expiry); analytics and
           trend; applications for own jobs (status, notes, interview); resume
           PDF stream for own applicants; notifications.
Admin:     Stats and trends; list/toggle/delete users; patch user role; list and
           moderate jobs (toggle status, set status, delete); companies delete;
           all applications; audit logs (paginated).

REPOSITORY STRUCTURE
────────────────────
backend/src/
  config/       Environment validation, logger, Prometheus metrics, security
                (CORS, Helmet, HPP, rate limit presets).
  constants/    Roles, application statuses, cookie names, etc.
  controllers/  HTTP handlers wired from routes.
  middlewares/  Auth, CSRF, validation, audit, idempotency, sanitize, errors.
  models/       Mongoose schemas (User, Job, Application, Company, etc.).
  queues/       BullMQ email queue; direct-send fallback when no REDIS_URL.
  repositories/ Thin data-access wrappers around models.
  routes/       Express routers; v1 aggregated under routes/v1/index.js.
  services/     Business logic (auth, jobs, applications, admin, etc.).
  utils/        JWT, cookies, mail, token blacklist, API errors/responses, etc.
  validations/  Zod schemas for request bodies and queries.
  workers/      emailWorker.js, jobAlertDigestWorker.js (BullMQ consumers).

frontend/src/
  api/          Axios apiClient (withCredentials) and domain API modules.
  components/   Shared UI, layout pieces, resume viewer, route guards.
  context/      AuthProvider (session restore, login, logout).
  features/     Larger UI slices by area (e.g. recruiter panels).
  hooks/        Reusable React hooks.
  layouts/      AppLayout shell (nav, mobile sheet).
  lib/          Query keys, auth constants, helpers.
  pages/        Route-level pages (home, jobs, dashboards, admin, legal).

Root: package.json runs concurrently backend:dev + frontend:dev (see scripts);
scripts/wait-for-api.cjs gates the web dev server until the API is up.

DATABASE MODELS
───────────────
User:         fullName, email, password (bcrypt cost 12), role (candidate|
              recruiter|admin), profile fields (headline, about, phone, location,
              website, skills), experience[], projects[], education[],
              profileImageUrl, resumeUrl/resumePublicId/resumeFileName/
              resumeSize/resumeUploadedAt, refreshToken (server-only; unset on
              logout), isActive, password reset fields, savedJobs[].
Job:          title, description, location, employmentType, experienceLevel,
              minSalary/maxSalary, skills[], company ref, postedBy, isActive,
              isDraft, expiresAt; timestamps.
Application:  job ref, candidate ref, coverLetter, status (APPLICATION_STATUSES),
              recruiterNotes, interview (scheduledAt, timezone, durationMinutes,
              notes, status); unique index job+candidate; timestamps.
Company:      name, website, location, description, logoUrl, createdBy; timestamps.
Notification: user ref, type (APPLICATION_RECEIVED|APPLICATION_STATUS_CHANGED|
              INTERVIEW_SCHEDULED|JOB_ALERT_MATCH|GENERAL), title, message, link,
              read, readAt, meta; timestamps.
JobAlert:     user ref, keywords[], location, employmentType, salaryMin,
              frequency (IMMEDIATE|DAILY|WEEKLY), isActive, lastSentAt; timestamps.
AuditLog:     userId, action enum (create|update|delete|login|logout|register|
              resume_viewed), resourceType, resourceId, changes, ip, userAgent;
              TTL index on createdAt from AUDIT_LOG_TTL_DAYS (default 365).

SECURITY
────────
- Access JWT (default 15m, JWT_ACCESS_EXPIRES); stored in memory on SPA.
- Refresh JWT in httpOnly cookie (default 7d, JWT_REFRESH_EXPIRES); path
  /api/v1/auth; rotation + reuse detection; $unset refresh hash on logout.
- CSRF double-submit for most POST/PUT/PATCH/DELETE under /api (exemptions in
  middlewares/csrf.js include login, register, refresh, logout, forgot-password,
  reset-password, csrf-token).
- Bcrypt hash on User save with salt rounds 12 (see models/User.js).
- Rate limits when RATE_LIMIT_ENABLED or production (see app.js and security.js).
- Helmet + CSP; CORS from CLIENT_URL; input sanitization; Zod validation;
  RBAC and resource ownership in middlewares/authorize.js.
- Audit log TTL on AuditLog.createdAt.

EMAIL & QUEUE
─────────────
Queue: BullMQ queue name "email" when REDIS_URL is set (ioredis connection).
Workers (from backend/package.json scripts):
  npm run worker:email
  npm run worker:job-alert-digest
Typed jobs in emailQueue.addEmailJob include:
  JOB_ALERT, APPLICATION_RECEIVED, APPLICATION_STATUS_CHANGED, INTERVIEW_SCHEDULED
(queue name = type). Defaults: attempts 3, exponential backoff delay 5000 ms.
Fallback: Direct nodemailer send in-process when Redis queue is unavailable
(for implemented types in emailQueue.js).

QUICK START
───────────
1. cd backend && cp .env.example .env && edit required keys
2. cd backend && npm install && (optional) same for frontend from frontend/
3. From repo root: npm install (installs root devDependencies: concurrently, wait-on)
4. npm run dev — runs backend:dev and frontend:dev (see root package.json)

Default ports in backend/.env.example: API PORT=5000. Vite dev server: 5173
(frontend/vite.config.js server.port). Frontend proxy /api uses VITE_BACKEND_PORT
or VITE_BACKEND_URL (see vite.config.js).
